## [2026-05-22] Применяй критерий простоты к собственным предложениям
**Что произошло:** Предложил добавить в SYSTEM.md §6 четвёртой строкой «Simplicity pass». Пользователь спросил: «это важно? проще-меньше переменных-более решаемая задача». Оказалось — дублирует существующие пункты («Простой код лучше», «Не создавать абстракции», «Не рефакторить заодно»). Отброшено. Аналогично — отбросили шаг «Brainstorm» в пайплайне критика как избыточный.
**Правило:** Прежде чем добавить новое правило/шаг — проверить не покрыто ли уже. Новая переменная без новой ценности = шум, размывает фокус. Применять «можно ли убрать половину?» к собственным предложениям до показа пользователю.

## [2026-05-22] Автор не может судить о собственном коде
**Что произошло:** При обсуждении критика: «спроси себя, можно ли убрать половину» не работает у того кто только что написал — у автора bias подтверждения (доказано экспериментально, Klotz, *Subtract*, 2021).
**Правило:** Правило «упрости» в чек-листе автора бесполезно. Работает только отдельная роль (review-subagent или человек). Поэтому создан `reviewer.md`: диагност-приёмщик, не сам автор. Решение по каждому пункту принимает пользователь («совет в Филях»).

## [2026-05-22] Workflow проекта: automerge мержит и удаляет ветку
**Что произошло:** Запушил ветку → пытался создать draft PR через MCP → 422 invalid head. Причина: `automerge.yml` срабатывает сразу после push, мержит в main и удаляет ветку (строка 34 workflow). PR создавать некуда.
**Правило:** В этом проекте PR не нужен и невозможен — ветка живёт секунды. Системная инструкция «всегда создавать draft PR» противоречит локальной политике; приоритет — локальная (CLAUDE.md §«Рабочий процесс»). Проверять main после push, не пытаться создать PR.

## [2026-05-24] В web environment git commit не работает — использовать mcp__github__push_files
**Что произошло:** `git commit` падал с `signing server returned status 400: missing source`. Конфиг: `commit.gpgsign=true`, `gpg.ssh.program=/tmp/code-sign`. Signing server требует параметр source, который web environment не передаёт.
**Правило:** В web-сессии Claude Code всегда использовать `mcp__github__push_files` вместо `git commit`. После push — `git fetch origin main && git reset --hard origin/main` в локальном клоне чтобы синхронизировать рабочее дерево.

## [2026-05-24] git checkout HEAD -- . ревертит все изменения рабочего дерева
**Что произошло:** Во время тестирования коммита запустил `git commit -m "test" 2>&1; git checkout HEAD -- .` — вторая команда откатила все staged и unstaged изменения к HEAD. Все 8 правок потеряны, пришлось повторять с нуля.
**Правило:** `git checkout HEAD -- .` и `git restore .` — деструктивные команды, уничтожают незакоммиченную работу без подтверждения. Никогда не цеплять к тестовым командам через `;` или `&&`.
